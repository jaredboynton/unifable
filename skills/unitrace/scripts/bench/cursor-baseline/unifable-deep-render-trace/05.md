I'll trace the trace-rt pipeline from entry point through explore, rehydrate, and markdown rendering.
## Overview

`trace-rt` is a **two-phase pipeline**: host-driven **explore** (read-only codebase gathering) followed by **submit** (structured synthesis + host rehydration + markdown rendering). The shell wrapper orchestrates prep and I/O; `realtime-trace.mjs` runs the model phases.

Default entry: `unitrace.sh` â†’ `trace-rt.sh` â†’ `realtime-trace.mjs`.

```mermaid
flowchart TD
  Q[Question] --> TR[trace-rt.sh]
  TR --> MAP[map.mjs repo map prefetch]
  TR --> PROMPT[Build explore + submit prompts]
  PROMPT --> RT[realtime-trace.mjs]
  RT --> EXP[Explore phase]
  EXP --> NAV[nav: rt-explore-nav.mjs]
  EXP --> AG[agentic fallback: explore_exec loop]
  NAV --> CACHE[filesRead + readCache]
  AG --> CACHE
  CACHE --> PKT[buildSubmitPacket + READ INDEX]
  PKT --> SUB[Submit phase]
  SUB --> DAEMON[runDaemonPointerSubmit]
  SUB --> SESS[runSubmitPhase session fallback]
  DAEMON --> REHY[rehydratePointerSubmit]
  SESS --> REHY
  REHY --> VAL[validateTraceObject]
  VAL --> REN[renderTraceStructured]
  REN --> OUT[out.md]
```

## 1. Shell wrapper: question â†’ run directory â†’ Node driver

**Files:** `skills/unitrace/scripts/trace-rt.sh`, `skills/unitrace/scripts/unitrace.sh`

`trace-rt.sh` takes the question as arguments, sets up an isolated run under `~/.cache/explore/runs/<run-id>/`, and launches Node.

Key prep steps:

4. **Post-process** â€” On success, writes `out.md`, `structured.json`, `raw`, sets `done`. If `UNITRACE_WIRE_FORMAT=1`, runs `explore_hydrate_trace_output` (wire token expansion); the default structured path skips that.

## 2. Explore phase: gather grounded evidence

**Files:** `skills/unitrace/scripts/realtime-trace.mjs`, `lib/rt-explore-nav.mjs`, `lib/rt-map-seed.mjs`, `lib/rt-tools.mjs`, `search-fast.mjs`

`runStructuredTrace()` in `realtime-trace.mjs` maintains:

- `filesRead` â€” set of relative paths actually read
- `readCache` â€” numbered excerpts (`123|line content`) per file, with pinned vs recent layers via `makeReadTracker()`

### Default mode: `nav` (host-driven micro-agent)

`dispatchExplore()` routes to `runExploreNav()` when `UNITRACE_RT_UNITRACE_MODE=nav` (default):

1. **Explicit seeds** â€” `seedExploreReads()` (`rt-map-seed.mjs`): question-named scripts, map line ranges, trace-specific targets (e.g. questions mentioning `trace-rt` auto-seed `trace-rt.sh` + `realtime-trace.mjs`).

2. **Host seed** â€” `search-fast.mjs` `retrieveCandidates()`: combined ripgrep â†’ AST hydrate â†’ pinned reads.

3. **Follow-ups** â€” Header reads, dependency tracing (imports/`source` lines), symbol usage greps.

4. **Navigator round(s)** â€” 8 parallel `gpt-realtime-mini` calls via `daemonAskBatch()` return `{ grep_terms, read_paths, done }`. Host executes greps + `toolReadRange()` â€” mini proposes, host reads.

5. **Fail-open** â€” Returns `null` if daemon unavailable and no seeds â†’ falls back to **agentic** `explore_exec` loop on the live Realtime WebSocket.

  if (mode !== "nav" && mode !== "hybrid") { ... agentic/daemon ... }
  const navStats = await runExploreNav({ workspace, question, mapBlock, filesRead, readCache, onRead, ... });
  if (!navStats) {
    toolLog.push("phase explore_mode=nav_failopen->agentic");
    return runExplorePhaseSession(await ensureSession(), args);
  }
  ...
}
```

### Agentic fallback

`runExplorePhaseSession()` opens a Realtime session, sends the explore prompt, and loops on `explore_exec` tool calls (`rt-tools.mjs` â†’ `grep`, `read`, `batch_read`, etc.). Stops when `shouldStopExplore()` hits read/tool-call caps.

After explore, explore conversation items are **pruned** from the socket (`session.pruneItems`) so submit gets a fresh context.
