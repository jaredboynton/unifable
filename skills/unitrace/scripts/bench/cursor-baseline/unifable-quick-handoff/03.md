## Handoff chain

`unitrace.sh` does not talk to the Realtime API itself. It is a thin default wrapper that sets trace-specific reasoning defaults and **replaces itself** with `trace-rt.sh`, which builds prompts/run state and invokes the Node realtime tracer.

```
unitrace.sh  →  trace-rt.sh  →  realtime-trace.mjs  →  RtAgentSession / daemon pool  →  Realtime WebSocket (gpt-realtime-2)
```

## 1. `unitrace.sh` — defaults + delegate

```12:15:skills/unitrace/scripts/unitrace.sh
exec env \
  UNITRACE_RT_UNITRACE_REASONING_EFFORT="${UNITRACE_RT_UNITRACE_REASONING_EFFORT:-none}" \
  UNITRACE_RT_SUBMIT_REASONING_EFFORT="${UNITRACE_RT_SUBMIT_REASONING_EFFORT:-low}" \
  "$SCRIPT_DIR/trace-rt.sh" "$@"
```

What it does:

- Resolves its own directory.
- Sets explore reasoning to **`none`** (omit + steer) and submit reasoning to **`low`** unless already set in the environment.
- **`exec`** into `trace-rt.sh` with all CLI args unchanged (the quoted question).

So the only behavior `unitrace.sh` adds over calling `trace-rt.sh` directly is those two env defaults.

## 2. `trace-rt.sh` — shell orchestrator → Node entry

This is where the actual handoff to the realtime tracer happens.

**Preflight & run layout**

- Sources `explore-hydrate.sh` for optional post-trace output hydration.
- Blocks recursive use when `UNITRACE_INSIDE_TRACE_DAEMON=1`.
- Validates args (one quoted question, no flags after it).
- Requires `node` and Codex OAuth at `~/.codex/auth.json`.
- Creates an isolated run dir under `~/.cache/explore/runs` (or `UNITRACE_RUNS_DIR`).

**Prompt assembly**

- Builds an **explore prompt** (`UNITRACE_PROMPT`) instructing `explore_exec`-only exploration.
- Builds a **submit prompt** (`SUBMIT_PROMPT`) for structured `submit_trace`.
- Optionally runs `map.mjs` (default mode `tandem`) and injects a compact repo map into the explore prompt.

**Invocation of the realtime tracer**

```378:397:skills/unitrace/scripts/trace-rt.sh
RT_ARGS=(
  --prompt-file "$PROMPT_FILE"
  --map-file "$MAP_FILE"
  --question "$QUESTION"
  --workspace "$WORKSPACE"

...
node "$SCRIPT_DIR/realtime-trace.mjs" "${RT_ARGS[@]}" || trace_status=$?
```

It passes file paths (not inline prompts), workspace, model (`gpt-realtime-2` by default), Codex auth path, and a frames NDJSON log. On success it writes `out.md`, `structured.json`, and status metadata.

## 3. `realtime-trace.mjs` — two-phase Realtime pipeline

**CLI entry (`main`)**

```1233:1293:skills/unitrace/scripts/realtime-trace.mjs

  const promptFile = argValue("--prompt-file");
  ...
  const explorePrompt = readFileSync(promptFile, "utf8");
  const submitInstructions = submitPromptFile ? readFileSync(submitPromptFile, "utf8") : "";
  ...

```

Reads prompt files from `trace-rt.sh`, then runs **`runStructuredTrace`** (or `runWireStructuredTrace` if `--wire 1`).

**Explore phase (`dispatchExplore`)**

Default mode is **`nav`** (`UNITRACE_RT_UNITRACE_MODE`, set in `realtime-trace.mjs`):

```587:618:skills/unitrace/scripts/realtime-trace.mjs

  const mode = UNITRACE_RT_UNITRACE_MODE;
  if (mode !== "nav" && mode !== "hybrid") {
    ...
    return runExplorePhaseSession(await ensureSession(), args);
  }
  const navStats = await runExploreNav({ ... });
  if (!navStats) {
    ... // fail-open to agentic explore_exec loop
    return runExplorePhaseSession(await ensureSession(), args);
  }
```

- **`nav` (default):** `runExploreNav` in `lib/rt-explore-nav.mjs` — host-side seeding + parallel `gpt-realtime-mini` navigators over the warm **`rtinferd` daemon pool** (`daemonAskBatch`). The host greps/reads files; mini only proposes paths/terms.
- **`agentic`:** full-model `explore_exec` loop over Realtime (daemon first, live session fallback).
- **`hybrid`:** nav plus optional one-turn agentic top-up.

**Live Realtime session (`ensureSession`)**

When a live WebSocket is needed:

```1003:1012:skills/unitrace/scripts/realtime-trace.mjs
const ensureSession = async () => {
  if (session) return session;
  session = new RtAgentSession({ model, authPath, framesPath });
  await session.connect();
  ...
  return session;
};
```

`RtAgentSession` (`lib/rt-agent-session.mjs`) wraps `RealtimeConnection` from `lib/realtime_client.mjs` — Codex OAuth token refresh, WebSocket connect to `api.openai.com/v1/realtime`, prewarm, hot-socket reuse, and context pruning before submit.

**Submit phase**

After explore, `buildSubmitPacket` assembles evidence; submit goes through:

1. **Daemon pool** (`runDaemonPointerSubmit` → `daemonAsk`) when `UNITRACE_RT_DAEMON=1` (default) — warm `gpt-realtime-2` over loopback `rtinferd`.
2. **Live session fallback** — `runSubmitPhase` / `askStructured` on the Realtime WebSocket if daemon misses or validation fails.

Pool warming starts concurrently with explore:

## 4. Transport layer — `realtime_client.mjs`

Low-level Realtime client:

- Reads/refreshes Codex OAuth from `~/.codex/auth.json`.
- Opens RFC6455 WebSocket to OpenAI Realtime API.
- Exports `RealtimeConnection`, `askStructured`, reasoning config helpers used by trace/search/websearch.

The daemon path (`lib/daemon-client.mjs` → `rtinferd`) reuses the same Realtime protocol but keeps warm sockets in a process pool so explore/nav/submit batches avoid per-call connect+handshake.

## Summary

| Layer | File | Role |
|-------|------|------|
| User entry | `unitrace.sh` | Sets explore=omit, submit=low; `exec trace-rt.sh` |
| Shell runner | `trace-rt.sh` | Auth check, map prefetch, prompts, run dirs; calls `realtime-trace.mjs` |
| Trace orchestrator | `realtime-trace.mjs` | Explore (nav/agentic) + submit; daemon-first, session fallback |
| Explore fast path | `lib/rt-explore-nav.mjs` | Host seed + mini navigators via daemon pool |
| Session wrapper | `lib/rt-agent-session.mjs` | Hot Realtime socket, prewarm, prune, reconnect |
| WebSocket client | `lib/realtime_client.mjs` | Codex OAuth + Realtime API |
| Daemon IPC | `lib/daemon-client.mjs` | Borrow warm Realtime sockets from `rtinferd` |

Documented supported path (from `skills/unitrace/scripts/AGENTS.md`):

> `unitrace.sh` → `trace-rt.sh` → `realtime-trace.mjs`

The “realtime tracer” is **`realtime-trace.mjs` and its lib stack**; `unitrace.sh` only forwards to `trace-rt.sh` with tuned defaults.
