Tracing how `unitrace.sh` delegates to the realtime tracer by following the shell entrypoint and its script calls.
Reading `realtime-trace.mjs` entry point and main orchestration flow:
`unitrace.sh` does not talk to the Realtime WebSocket itself. It is a thin shell wrapper that sets trace defaults and **replaces itself** with `trace-rt.sh`, which prepares prompts/run state and invokes `realtime-trace.mjs` — the Node program that runs the two-phase Realtime trace.

## Call chain

```text
unitrace.sh
  └─ exec env … trace-rt.sh "$@"
       └─ node realtime-trace.mjs --prompt-file … --question …
            └─ runStructuredTrace() → explore → submit → markdown out
```

## Step 1: `unitrace.sh` → `trace-rt.sh`

`unitrace.sh` only sets reasoning-effort defaults and hands off via `exec`:

```12:15:skills/unitrace/scripts/unitrace.sh
exec env \
  UNITRACE_RT_UNITRACE_REASONING_EFFORT="${UNITRACE_RT_UNITRACE_REASONING_EFFORT:-none}" \
  UNITRACE_RT_SUBMIT_REASONING_EFFORT="${UNITRACE_RT_SUBMIT_REASONING_EFFORT:-low}" \
  "$SCRIPT_DIR/trace-rt.sh" "$@"
```

- **Explore:** `UNITRACE_RT_UNITRACE_REASONING_EFFORT=none` (omit reasoning + steer on user turns)
- **Submit:** `UNITRACE_RT_SUBMIT_REASONING_EFFORT=low`
- **`exec`:** same PID; all CLI args (`"$@"`) pass through unchanged
- No map generation, auth checks, or Node invocation here

## Step 2: `trace-rt.sh` prepares the run and invokes Node

`trace-rt.sh` is the shell orchestrator. Before calling Node it:

1. **Preflight:** requires `node`, Codex OAuth at `~/.codex/auth.json` (or `UNITRACE_CODEX_AUTH_PATH`)
2. **Run layout:** isolated run dir under `UNITRACE_RUNS_DIR` or `~/.cache/explore/runs` (`out.md`, `err.log`, `structured.json`, `frames.ndjson`, etc.)
3. **Repo map:** optional prefetch via `map.mjs` (`UNITRACE_MAP_MODE`, default `tandem`)
4. **Prompts:** writes explore + submit prompt files (explore instructions, repo map, question; submit schema instructions)
5. **Child env:** sets `UNITRACE_WORKSPACE` and `UNITRACE_INSIDE_TRACE_DAEMON=1` (blocks recursive trace from inside the tracer)

The handoff to the realtime tracer is this `node` invocation:

```378:397:skills/unitrace/scripts/trace-rt.sh
RT_ARGS=(
  --prompt-file "$PROMPT_FILE"
  --map-file "$MAP_FILE"
  --question "$QUESTION"
  --workspace "$WORKSPACE"
  --out "$TMP_OUT"
  --raw "$TMP_RAW"
  --err "$ERR_FILE"
  --model "$MODEL"
  --auth-path "$CODEX_AUTH"
  --frames "$RUN_DIR/frames.ndjson"
)

RT_ARGS+=(--submit-prompt-file "$SUBMIT_PROMPT_FILE" --structured-out "$STRUCTURED_JSON")
…
node "$SCRIPT_DIR/realtime-trace.mjs" "${RT_ARGS[@]}" || trace_status=$?
```

On success it moves temp output to `$RUN_DIR/out.md`, writes `done`, and prints the trace plus run metadata. On failure it surfaces stderr from `realtime-trace.mjs`.

Default model: `gpt-realtime-2` (`UNITRACE_RT_MODEL`).

**Phase A — Explore (`dispatchExplore`)**

Default mode is **`nav`** (`UNITRACE_RT_UNITRACE_MODE`, from env or default):


- **`nav`:** host seeds reads, then 8× parallel `gpt-realtime-mini` navigators (`lib/rt-explore-nav.mjs`) propose grep/read paths; host hydrates into `readCache`
- **`agentic`:** full `explore_exec` loop over a live Realtime session (`runExplorePhaseSession`) or daemon (`runExplorePhaseDaemon`)
- **`hybrid`:** nav plus optional one-turn agentic top-up

Explore reasoning comes from `UNITRACE_RT_UNITRACE_REASONING_EFFORT` (set to `none` by `unitrace.sh`).

**Phase B — Submit**

1. `buildSubmitPacket()` assembles question, files read, excerpts/pointer index, submit instructions
2. Prefer **`runDaemonPointerSubmit()`** on warm `rtinferd` pool (`lib/daemon-client.mjs`, namespace `"trace"`, synth model `gpt-realtime-2`, reasoning `low`)
3. Fail-open to **`runSubmitPhase()`** on a live Realtime WebSocket (`lib/rt-agent-session.mjs`, `lib/realtime_client.mjs`)
4. **`renderTraceStructured()`** turns structured JSON into final markdown; pointer rehydration in `lib/rt-rehydrate-submit.mjs` / `lib/rt-pick-passages.mjs`

## What `unitrace.sh` actually contributes

| Layer | Role |
|--------|------|
| `unitrace.sh` | Default reasoning knobs + `exec` into `trace-rt.sh` |
| `trace-rt.sh` | Auth/preflight, run dirs, map prefetch, prompt assembly, `node realtime-trace.mjs` |
| `realtime-trace.mjs` | Realtime explore + submit, daemon pool, markdown rendering |

Calling `trace-rt.sh` directly skips only the two env defaults from `unitrace.sh`; the Realtime path is the same.
