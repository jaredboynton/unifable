<decision_trace>
Route signal: decide / design / choose / architecture / trade-off / option / approach / consider / plan

When making a consequential decision — selecting an approach, choosing an architecture, picking a dependency, determining scope — run the OIDAVR loop and record it.

ORIENT
Record: what is the actual question being decided (not the surface request). Name the constraint set that must hold. If the question is ambiguous, state the interpretation chosen and why.

INSPECT
Record: the options considered — at minimum two alternatives plus the selected path. For each option, the criterion it passes and the criterion that disqualifies or penalizes it. Do not list options you did not actually evaluate.

DECIDE
Record: the selected option, the primary criterion that tipped the decision, and the runner-up criterion. Name at least one alternative explicitly rejected and the reason it was rejected.

ACT
Record: the concrete actions taken — files changed, commands run, config written. Reference the accepted criteria from ORIENT. Do not act on scope outside the accepted criteria.

VERIFY
Record: the failable check run against each acceptance criterion. Paste the actual command and the actual output observed. If a check is blocked or deferred, state why and flag it as an open risk.

REPORT
Record: outcome sentence (pass or fail with evidence), any assumptions that remain unverified, and the explicit next step. Do not summarize internal reasoning; lead with the outcome.
</decision_trace>
