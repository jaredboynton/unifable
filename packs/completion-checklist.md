# Completion Checklist

Complete before marking any normal or deep task done. Each item must be checked or explicitly marked N/A with a reason.

- [ ] **Durable decisions recorded.** Every consequential decision (approach chosen, alternative rejected, constraint accepted) is written into a decision trace or note — not only in the conversation transcript.
- [ ] **Verified commands captured.** Every command confirmed to work is recorded verbatim with its output, in a note or the goals ledger. Commands not rerun in this session are flagged.
- [ ] **TDD / tests noted.** Any test written, modified, or run is identified. Pre-existing tests that must not be weakened are named. No test was deleted, skipped, or softened without an equivalent replacement.
- [ ] **Stale notes removed.** Any prior note, comment, or checklist entry that no longer reflects current state has been updated or deleted. No contradictory claims exist between current notes and current code.
- [ ] **Index links updated.** If the project maintains a notes index or decision-trace index, new entries have been appended. No orphaned files exist outside the index.
- [ ] **Back-links added.** If this task produced a result referenced by another task, the referencing entry has been updated to point to this result.
- [ ] **Residual risks listed.** Any assumption that was not verified in this session, any deferred check, and any known open edge case is named explicitly. Each residual risk has an owner or a next-step action.
